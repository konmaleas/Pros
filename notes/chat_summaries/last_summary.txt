## Critical Bug Found - KeyError 'processed_lines'

### The Problem
**Location:** `pattern_analyzer.py` line 205, in `_find_stable_anchor_lines()`

```python
# Line 136-137 in analyze_all_files() - STORES:
all_file_data[str(file_path)] = {
    'filtered_lines': filtered_lines,        # ← Key name is 'filtered_lines'
    'chunk_positions': filtered_frequency_data['chunk_line_positions']
}

# Line 205 & 216 in _find_stable_anchor_lines() - ACCESSES:
for line in first_file['processed_lines']:  # ← Expects 'processed_lines'
```

**Root Cause:** Dictionary key mismatch - storing `'filtered_lines'` but accessing `'processed_lines'`

### The Fix Needed
Add `processed_lines` to the dictionary in `analyze_all_files()`:

```python
all_file_data[str(file_path)] = {
    'processed_lines': processed_lines,    # ← ADD THIS LINE
    'filtered_lines': filtered_lines,
    'chunk_positions': filtered_frequency_data['chunk_line_positions']
}
```

This is line 136-137 in `/mnt/project/pattern_analyzer.py`
