# PDF Title Analysis Pipeline - Project Summary

## Project Goal
Build a composable pipeline to extract and analyze title patterns from PDF files for auto-completion system. Handles dynamic title formats like `4005-RES-VAP-DWG-233-IC-07020-0` where some chunks are static (RES, VAP, DWG) and others are dynamic (233, 07020).

## Core Challenge Solved
**Circular Dependency Problem**: Original code had FrequencyBasedFilter needing frequency data from TitleSearch, which needed filtered text from FrequencyBasedFilter. Pipeline architecture breaks this cycle.

## Pipeline Architecture (4 Stages)

### Stage 1A: TextExtractor
```python
@dataclass
class TextExtractor:
    def extract_text(self, pdf_path: str) -> str
```
- Pure PDF → Raw text string extraction
- Uses PyMuPDF, no filtering/processing

### Stage 1B: TextProcessor  
```python
@dataclass
class TextProcessor:
    def process_text(self, raw_text: str) -> List[str]
```
- Raw text → Clean line-split text
- Character normalization, line splitting
- Uses your exact `clear_file_()` logic

### Stage 2: FrequencyAnalyzer
```python
@dataclass  
class FrequencyAnalyzer:
    def analyze_text_frequencies(self, lines: List[str]) -> FrequencyData
```
- **Eliminates ALL redundancy** from original classes
- Consolidates frequency counting from both TitleSearch and FrequencyBasedFilter
- Tracks frequencies AND line positions in one pass
- Provides `get_filtering_data()` for Stage 3

### Stage 3: AdaptiveFilter
```python
@dataclass
class AdaptiveFilter:
    def filter_text(self, lines: List[str], freq_data: FrequencyData) -> List[str]
```
- Takes pre-calculated frequency data (breaks circular dependency)  
- Preserves your exact filtering logic (76-threshold, pattern detection)
- Uses spaCy for grammatical analysis
- No redundant frequency counting

### Stage 4: PatternAnalyzer
```python
@dataclass
class PatternAnalyzer:
    def analyze_file_patterns(self, filtered_lines, filename, chunk_positions) -> FileAnalysisResult
```
- Template-based approach for handling dynamic vs static chunks
- Uses your template file identification function
- Flexible anchor selection (Cluster → Density → Fallback strategies)
- Bidirectional distance mapping (positive = after anchor, negative = before)
- Per-file output for auto-completion system

## Key Architectural Decisions

### Template File Strategy
- You have a function that identifies the "complete" template file
- Template file becomes reference for all other files  
- Single anchor point preferred but flexible for multiple if needed
- Dynamic chunks identified by positional differences from template

### Anchor Selection (3 Strategies)
1. **Cluster Strategy**: Groups chunks within 100 lines, finds largest cluster
2. **Density Strategy**: Finds positions with highest concentration of other chunks
3. **Fallback Strategy**: Most frequent chunk or first occurrence

### Data Flow (Eliminates Circular Dependency)
```
PDF → TextExtractor → TextProcessor → FrequencyAnalyzer → AdaptiveFilter → PatternAnalyzer
```

## What Was Eliminated/Replaced

### From TitleSearch:
- `line_chunks_nums()` → FrequencyAnalyzer.analyze_text_frequencies()
- `chunks_analysis()` → FrequencyAnalyzer.meets_operation_criteria()  
- Redundant text processing → Eliminated

### From FrequencyBasedFilter:
- `_count_word_frequencies()` → FrequencyAnalyzer.analyze_text_frequencies()
- `_calculate_noise_threshold()` → FrequencyAnalyzer.calculate_noise_threshold()
- Circular dependency issues → Eliminated
