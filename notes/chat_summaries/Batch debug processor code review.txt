Objective
Create batch_debug_processor.py to process 91 project directories (1,225 PDF files across 44 projects, 47 cases) and generate comprehensive debugging reports for the title extraction pipeline.
What We Built

batch_debug_processor.py - Initial version

Processes all cases from project_dirs.text

Infers titles from PDF filenames using regex
Runs MultiFilePatternAnalyzer on each case
Generates JSON and text reports

Auto-generated output directory structure: reports/batch_debug/v19/YYYYMMDD_HHMMSS/
CaseResult dataclass with metrics (PDFs, success/failure, processing time)
ProjectResults aggregation
Comprehensive error handling with full tracebacks
