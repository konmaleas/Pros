# Project Overview
Developing a text processing pipeline for PDF documents (architectural/engineering) to:
- Extract and filter meaningful content while removing noise
- Identify structural document patterns (title chunks like '4005-RES-VAP-DWG-233-IC-07020-0')
- Enable auto-completion based on consistent positioning patterns
- Detect dynamic vs static title chunks across files

# Architecture - 4-Stage Pipeline
## Stage 1A: TextExtractor
- Pure PDF → Raw text extraction using PyMuPDF
- No processing, single responsibility

## Stage 1B: TextProcessor  
- Raw text → Clean line-split text
- Character normalization (Greek Β → Latin B)
- Uses exact clear_file_() logic

## Stage 2: FrequencyAnalyzer
- Consolidates ALL frequency counting
- Provides chunk_line_positions, word_frequencies, noise_threshold
- Breaks circular dependency by providing data BEFORE filtering

## Stage 3: AdaptiveFilter
- Uses frequency data + clean lines → Intelligently filtered text
- 76-threshold, spaCy grammatical filtering
- Preserves structural terms, removes measurements

## Stage 4: MultiFilePatternAnalyzer
- Template-based approach for dynamic chunk detection
- Multi-anchor support for split title scenarios
- Bidirectional distance mapping
